================================================================================
SAMENVATTING VAN DE UITVINDING
================================================================================

Deze uitvinding betreft een intelligente databasescanner die automatisch 
Persoonlijk Identificeerbare Informatie (PII) detecteert in relationele 
databases (PostgreSQL, MySQL, SQL Server) met drie adaptieve scanmodi:

SCANMODI:
- FAST: 100 rijen per tabel, 2 parallelle workers (snelle compliance check)
- SMART: 300 rijen per tabel, 3 parallelle workers (gebalanceerde scan)  
- DEEP: 500 rijen per tabel, 3 parallelle workers (grondige analyse)

KERNFUNCTIONALITEIT:
De scanner analyseert automatisch database-schema's en berekent 
prioriteitsscores voor tabellen op basis van risico-indicatoren:
- User/Customer/Employee tabellen: 3.0× prioriteit (hoogste risico)
- Medical/Health/Patient tabellen: 3.0× prioriteit (AVG Artikel 9)
- Payment/Transaction tabellen: 2.8× prioriteit (financiële data)
- Overige tabellen: 1.5-2.5× prioriteit (basis PII)

NEDERLANDSE PII-DETECTIE:
Het systeem detecteert specifiek Nederlandse persoonsgegevens inclusief:
- BSN-nummers met validatie volgens officiële 11-proef algoritme: 
  (9×d₁ + 8×d₂ + 7×d₃ + 6×d₄ + 5×d₅ + 4×d₆ + 3×d₇ + 2×d₈ - 1×d₉) mod 11 = 0
- Nederlandse e-mailadressen (.nl domein)
- Nederlandse telefoonnummers (+31 6, 06 formaten)
- Nederlandse postcodes (1234 AB formaat)
- IBAN bankrekeningen (NL## formaat)

PARALLEL PROCESSING:
De scanner gebruikt Python's ThreadPoolExecutor voor efficiënte scanning:
- 2 workers voor FAST mode (minimale database-belasting)
- 3 workers voor SMART/DEEP modes (optimale doorvoer)
- Automatische result aggregatie en error handling

MARKTPOSITIE:
Deze uitvinding is specifiek ontwikkeld voor de Nederlandse markt met:
- UAVG compliance (Nederlandse implementatie van AVG/GDPR)
- BSN-detectie (uniek Nederlands identificatienummer)
- Nederlandse privacy autoriteit (Autoriteit Persoonsgegevens) richtlijnen
- Betaalbare oplossing voor MKB (€250/maand SaaS model)

TECHNISCHE IMPLEMENTATIE:
De scanner is geïmplementeerd in Python 3.11 met:
- psycopg2 voor PostgreSQL connectiviteit
- mysql.connector voor MySQL connectiviteit  
- pyodbc voor SQL Server connectiviteit
- concurrent.futures voor parallel processing
- regex-gebaseerde PII pattern matching
- GDPR artikel-mapping voor compliance rapportage

SCHAALBAARHEID:
De scanner schaalt lineair met:
- Database grootte: 10 tabellen → 10.000 tabellen
- Worker count: 2-3 parallelle threads (configureerbaar)
- Sample sizes: 100-500 rijen per tabel (mode-afhankelijk)
- Memory footprint: ~100MB voor 10.000 tabellen

BEVEILIGING:
- Geen data exfiltratie (alleen metadata en sample queries)
- Read-only database toegang (alleen SELECT statements)
- Encrypted connecties (SSL/TLS support)
- Audit logging van alle scan activiteiten
- Compliance met AVG Artikel 32 (beveiliging verwerking)

UITVOERINGSVORMEN:
1. SaaS model: Cloud-hosted scanner met multi-tenant architectuur
2. On-premise: Docker container deployment voor air-gapped networks
3. Hybrid: Lokale scanner met cloud-based rapportage

scanner te bieden die 95% goedkoper is dan enterprise oplossingen en specifiek
